## 1. 🔴 본업을 "임팩트 카드"로 격상

**문제**: 5년차 풀스택인데 Selected Projects가 전부 사이드/취미 프로젝트(메이플·로스트아크·포켓몬 키캡·키보드 테스터·이미지 에디터). 정작 본업(커머스·광학 플랫폼)이 비공개라 한 줄도 안 보임 → "게임/키보드 좋아하는 사람"은 전달되지만 "커머스 플랫폼을 5년 만든 사람"의 근거가 약함.

**방향**: 취미 프로젝트는 그대로 자산으로 두되, 경력 섹션을 비공개라도 **임팩트 카드**로 끌어올리기. 공개 가능한 범위(반올림·범위 표기)로라도 수치를 넣으면 신뢰도 급상승.

**작성 템플릿** (빈칸 채우기):

```
- 프로젝트/제품명(비공개 가능): __________
  - 한 줄 설명: __________ (무엇을 하는 서비스인가)
  - 규모/지표: __________ (예: 일 주문 N건, 사용자 N명, 데이터 N만 건 — 공개 가능 범위)
  - 내 역할: __________ (설계? 전체 개발? 어느 모듈?)
  - 스택: __________
```

**예시**:
> "셀러 어드민 (비공개) · 일 N건 주문을 처리하는 커머스 백오피스 · SvelteKit + Node, 검색/통계/정산 모듈 설계"

## 2. 🟡 프로젝트 설명: "기능 나열형" → "문제·결과형"

**문제**: 지금 설명이 기능 목록. "무엇을 풀었나 / 왜 만들었나 / 규모"가 없음.

**Before → After 예시**:

| 프로젝트 | 현재 (기능 나열) | 제안 (문제·결과) |
|----------|-----------------|------------------|
| Svelte Image Editor | "브라우저에서 동작하는 이미지 편집기 · 크롭 / 필터 / 캔버스" | "서버 없이 브라우저에서 끝나는 이미지 편집기 — Canvas 직접 제어로 무설치·무업로드 처리" |
| meso.gg | "메이플스토리 데이터 검색 서비스 · 풀텍스트 검색 프론트엔드와 데이터 스크랩 서버" | (라이브 서비스 → 규모 추가) "N만 건 데이터 풀텍스트 검색 · 스크랩 서버 + 검색 프론트" |

**규칙**: 라이브 서비스는 **사용자/데이터 규모**를 한마디 넣기. 패턴은 *"무엇을 + 차별점/성과"*.

**채울 정보**:
- meso.gg 데이터 규모: __________
- Pokémon Keycap 사용 맥락/특징: __________
- Keyboard Tester 차별점: __________
- LostArk Card Manager 규모/특징: __________

## 3. 🟡 plate-list 정리

**문제**: 가장 눈에 띄는 자리(meso.gg 바로 아래)에 **링크도 없고 "진행 중"인** 프로젝트. 클릭할 게 없어 김 빠짐. 현재 설명("SvelteKit + Node 풀스택 · 진행 중")은 정보량이 거의 0.

**선택지** (택1 이상):
- [ ] 순서를 목록 뒤로 내리기
- [ ] 최소한 GitHub repo 링크 연결 (`https://github.com/asdfghjkkl11/platelist`)
- [ ] "진행 중"이면 *무엇을 만드는 중인지* 한 줄 구체화 → 채울 내용: __________

## 4. 🟡 경력 bullet: "책임" → "성과"

**문제**: bullet이 책임 나열. "5년간 / 다수 / 전반 / 50+ 저장소" 같은 모호어·양적 표현이 많고 성과가 안 보임. ("50+ 저장소"는 양이지 성과가 아님 — 오히려 산만해 보일 수 있음.)

**Before → After 예시**:
> Before: "Java / Go 백엔드 서비스 개발, 50+ 사내 저장소에 기여"
> After: "결제·주문 시스템을 Flutter 앱에 연동해 모바일 전환율/안정성 개선" (가능하면 숫자)

**규칙**: 모호어(다수/전반/여러)를 **구체 명사 + 동사 + 숫자**로 교체.

**현재 4개 bullet 재작성** (`portfolio.ts` `experiences[0].points`):
1. __________
2. __________
3. __________
4. __________

## 5. ⚪ 추가 콘텐츠 (선택)

- [ ] **이력서(PDF) 다운로드** 버튼 — 채용 담당자가 가장 원하는 것. `public/`에 PDF 두고 Contact 섹션에 링크.
- [ ] **About 서사 한 문단** — 지금은 히어로 intro가 유일. "어떤 개발자가 되고 싶은지 / 무엇을 잘하는지" 짧은 단락 추가 고려.
- [ ] **연도 라벨 정리** — 2026이 4개. "운영 중 / 진행 중 / 2023" 식 상태 라벨이 더 명확. (`Project.year`를 상태 표현으로 확장하거나 별도 `status` 필드 추가)
- [ ] **영문 버전** — 영문 이름·JSON-LD는 있는데 본문은 한국어뿐. 해외/외국계 타겟이면 영문 토글 고려.
